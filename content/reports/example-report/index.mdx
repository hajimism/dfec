## はじめに

これはMDXで書かれたレポートのサンプルです。

<Callout type="info">
このレポートでは、カスタムコンポーネントの使用例も紹介しています！
</Callout>

## 特徴

MDXでは以下のようなことができます：

- **Markdown記法**が使える
- Reactコンポーネントが埋め込める
- コードブロックも美しく表示

```tsx
const hello = () => {
  console.log("Hello, World!");
};
```

<Callout type="warning">
カスタムコンポーネントは `components.tsx` に定義します。
</Callout>

## 画像の表示

### 標準的なMarkdown記法

MDXでは通常のMarkdown記法で画像を表示できます：

![Next.js Logo](/next.svg)

### Next.js Imageコンポーネント（最適化版）

カスタムコンポーネントを使えば、Next.jsのImageコンポーネントで最適化された画像表示も可能です：

<OptimizedImage src="/vercel.svg" alt="Vercel Logo" width={600} height={300} />

SVG画像も問題なく表示されます。publicフォルダに配置した画像は `/` から始まるパスで参照できます。
